我的Hugo个人博客搭建过程（二）

Jul 11, 2020
1 min read

第一篇讲了Hugo环境的搭建和一些小问题，接下来就总结一下后来部署到远程GitHub上的步骤

一、 在GitHub上的配置

  • 新建仓库
  • 注意名字是用户名小写形式 + github.io
  • 复制两段链接字段，备用

二、 新建静态页面

  • 1、 在根目录下输入 hugo命令，成功后hugo会在根目录下创建一个public文件夹

  • 2、 在配置文件中，将baseUrl改成 baseUrl="https://victoria-ai.github.io” （这里是我新建的仓库链接，配置时注意要修改成自己的）

三、与远程GitHub建立连接

  • 进入public文件夹，接下来是连续的git命令操作：

– git init (在public文件夹下初始化一个本地仓库)

– git add . （监控工作区，也就是根目录中的文件发生变化，提交给暂存区）

– git commit -m “要显示的提交日志” (将暂存区改动提交到本地版本库)

– git remote add origin https://github.com/Victoria-Al/victoria-ai.github.io.git

– git push -u origin master (这里会要求你输入GitHub的账户密码，将public里的文件push到仓库中)

四、访问部署后的Hugo个人博客

  • 在第一步时复制的github.io结尾的链接就是部署后的访问链接，以后可直接访问，因为现在它还是静态页面，之后可以自己尝试去配置和修改个人的博客网站

五、之后的提交博客文件操作

  • 1、 在根目录下用终端 键入hugo命令
  • 2 进入public文件夹，输入之前类似的提交命令（此时不用再初始化仓库和链接远程GitHub地址）

– git add .

– git commit -m “要显示的提交日志”

– git push -u origin master

  • 上面是我的配置文件，还没怎么配置。这里我选择的是直接在config.toml配置文件中修改theme和baseUrl的值，所以我在启动和新建静态页面时，不需要指定主题。如果没有先配置，则在之前的命令中要修改：

– hugo server -t m10c （第一次启动预览）

– hugo –theme=m10c –baseUrl="https://victoria-ai.github.io” （新建静态站点时）

以上就是我在搭建Hugo个人博客网站的过程，总结一下整个过程不是很难，主要是要细心一点，有时因为链接或者敲错命令会影响进度，所以一定要细心细心再细心！！ 这也是自己的第二篇文章了，接下来会更新自己对学习的内容的总结和理解，巩固知识； 目前正在Java的海洋中遨游 ！！
